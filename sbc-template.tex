\begin{abstract}
The paper describes the design and implementation of a modular smart wallet for Ethereum based on ERC-4337. This wallet is designed to be modular using the pattern multi-facet proxy, which means that users can add and remove modules as they see fit. The wallet is also designed to be extensible, which means that developers can create new modules that can be added to the wallet. The wallet is implemented in Solidity and is deployed on the Ethereum blockchain.
\end{abstract}

\section{Introdução}

No Ethereum, existem dois tipos distintos de contas: as {\textit{externally owned accounts}} (EOA) e as {\textit{contract accounts}} (CA). As EOAs são contas controladas por chaves privadas, enquanto as CAs são geridas por uma lógica de programação definida em um {\textit{smart contract}} (contrato inteligente). Toda transação realizada no Ethereum é assinada digitalmente com a chave privada da conta que iniciou a transação, sendo que esta deve ser, obrigatoriamente, uma EOA.\comment{REF}{Você precisa colocar referência em todos os parágrafos que você afirma coisas que não pode provar. No livro Mastering Ethereum, os autores Antonopoulos e Wood (2018) destacam:

\begin{quote}
    A palavra 'carteira' é usada para descrever algumas coisas diferentes no Ethereum. De maneira 
    genérica, uma carteira é um aplicativo de software que serve como a principal interface do usuário 
    para o Ethereum. A carteira controla o acesso ao dinheiro do usuário, gerenciando chaves e endereços,
    acompanhando o saldo e criando e assinando transações. Além disso, algumas carteiras Ethereum também 
    podem interagir com contratos, como tokens ERC20. De uma forma mais restrita, da perspectiva de um
    programador, a palavra carteira se refere ao sistema usado para armazenar e gerenciar as chaves do
    usuário. Toda carteira tem um componente de gerenciamento de chaves. Para algumas carteiras, isso é 
    tudo o que existe. Outras carteiras fazem parte de uma categoria muito mais ampla, que são interfaces 
    para aplicativos descentralizados baseados em Ethereum, ou DApps. Não há linhas claras de distinção 
    entre as várias categorias que são confundidas sob o termo carteira.\cite{MasteringEthereum}
\end{quote}


Aplicações descentralizadas (DApps) substituem a infraestrutura tradicional de back-end por smart contracts executados em blockchains, como o Ethereum. Entretanto, a adoção desses DApps enfrenta um desafio significativo: o nível de conhecimento técnico necessário para interagir com eles. Para interagir com um DApp, o usuário precisa, inicialmente, criar uma conta.

Ferramentas como a MetaMask simplificam o processo de criação de contas e interação com os DApps. A MetaMask age como uma ponte, permitindo que os usuários acessem o Ethereum e os DApps diretamente de seu navegador, através de uma extensão, sem a necessidade de operar um nó Ethereum completo. Esta carteira digital proporciona aos usuários a capacidade de gerenciar suas chaves privadas e endereços Ethereum, assinar transações e interagir diretamente com DApps.

Contudo, mesmo com a existência de ferramentas facilitadoras como a MetaMask, ainda há barreiras substanciais que podem impedir a adoção em massa de DApps e, de forma mais ampla, da blockchain Ethereum. A necessidade de gerenciar chaves privadas, por exemplo, é uma responsabilidade grande que pode ser intimidante para usuários comuns. Diferentemente de senhas convencionais, as chaves privadas não podem ser redefinidas se perdidas, e um backup adequado é vital.

Além disso, a consciência sobre segurança no espaço das criptomoedas é fundamental. Os usuários precisam estar constantemente alertas contra ameaças potenciais, como ataques de phishing, que podem levar ao roubo de suas chaves privadas e, consequentemente, de seus ativos digitais. A necessidade de tal vigilância constante pode ser desgastante e dissuadir novos usuários.

Outra consideração importante é a complexidade inerente ao espaço da blockchain. Embora provedores de carteiras como a MetaMask tenham feito grandes avanços na simplificação da interação com DApps, ainda existe uma curva de aprendizado significativa associada ao uso da tecnologia blockchain. Conceitos como endereços de blockchain, transações, gas fees e smart contracts podem ser difíceis de compreender para novos usuários, constituindo assim outra barreira à adoção em massa.

Em resposta a esses desafios inerentes, surgem esforços contínuos para aprimorar e simplificar a experiência do usuário no ambiente do Ethereum. Como o Ethereum é uma rede descentralizada e open source, propostas de melhorias, conhecidas como \textit{Ethereum Improvement Proposal} (EIPs), são constantemente formuladas pela comunidade. Uma proposta que se torna relevante para nós é a EIP-4337, também apelidada pela comunidade como \textit{Account Abstraction}.

Como ja citado anteriormente, as \textit{accounts} no Ethereum podem ser de dois tipos: \textit{Contract Accounts} (CAs) e \textit{Externally Owned Accounts} (EOAs). Enquanto uma EOA executa uma única operação por transação, uma CA pode realizar várias operações diferentes por meio de código em uma única transação.
